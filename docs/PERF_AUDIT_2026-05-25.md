## CPU hot loops

- [x] **(#C-xform) TransformSystem.cpp:69-77, 102-115** — was recursing into clean subtrees just to check if any descendant was dirty. **Landed:** added `kSubtreeDirty = 0x02` bit to `TransformComponent::flags`; system maintains it itself via a Pass 1 pre-pass that iterates the `<TransformComponent, HierarchyComponent>` join view and walks self-dirty entities' parent chains, setting `0x02` on each ancestor (early-out the moment an ancestor already has it). Pass 2 (the existing top-down recursion) now short-circuits when a node has neither `self-dirty`, nor `subtree-dirty`, nor an upstream `parentDirty` — the entire subtree is skipped. Game-side `tc->flags |= 0x01` call sites unchanged (~20 sites across engine + apps) — the subtree bit is system-managed. Cost analysis: flat scenes (no hierarchy) pay essentially zero for Pass 1 because the join view is empty; hierarchical scenes pay O(dirty × distinct-ancestors) for Pass 1 in exchange for O(1) skip of clean subtrees in Pass 2. **Tests pinning the optimisation:** (1) "clean subtree is fully skipped on second update" poisons world matrices after the first update and asserts they survive the second update — proves the recursion never enters the clean subtree; (2) "dirty leaf propagates subtree-dirty up the parent chain" pins the partial-dirty case with a clean sibling that must stay poisoned; (3) "subtree-dirty bit is a system-managed flag" pins the policy that game code shouldn't set 0x02 directly and that the system tolerates spurious sets. **perf_smoke (flat scene, 702 entities): TransformSystem mean 0.057 ms — within noise of prior baseline (no regression on the fast path).** 724 unit + 22 screenshot tests pass. *low.*
- [x] **(#C-xform-trs) TransformSystem.cpp:15-21** — `composeLocal` was `translate * mat4_cast(quat) * scale` (~176 muls/call counting all three glm temporaries).  **Landed:** direct TRS construction — `Mat3 rot = mat3_cast(q); return Mat4(rot[0]*s.x, rot[1]*s.y, rot[2]*s.z, (pos, 1))`.  Drops to ~29 muls/call, no Mat4 temporaries on the stack.  Audit projected ~50% TransformSystem saving; measured 26% on the dirty-all worst case (the audit muls count missed the dirty-flag check + view iteration overhead that share the per-entity envelope).  *low.*
- [x] **(#C-xform-cache) TransformSystem.cpp:51-72** — was `reg.has<WorldTransform>` + `reg.get<WorldTransform>` (inside the old `setWorldMatrix` helper) + `reg.emplace<WorldTransform>` = three sparse-set lookups per child.  **Landed:** single `reg.get<>()` at the top of each entity's processing, treat `nullptr` as missing-world, pass the cached pointer into the new `writeWorldMatrix` helper which either writes in place or emplaces.  Audit projected 0.02 ms / 1000 entities; landed with composeLocal in the same commit so the share of the combined 26% mean-time win can't be isolated, but the `wtc != nullptr` branch in the not-dirty-but-subtree-dirty path also removes a redundant get — that branch fires on every interior node of a partially-dirty hierarchy.  *trivial.*
- [x] **(#C-frustum) Frustum.h:53-59** — `glm::length(Vec3(p)) / division` per plane → `1 fdiv + 4 fmul` per plane via `invLen = 1 / sqrt(lenSq); p *= invLen`. On Cortex-A78 fdiv is ~7c vs fmul ~3c, so 6 sqrt + 24 fdiv per Frustum became 6 sqrt + 6 fdiv + 24 fmul (~33% cheaper per ctor). Also drops the `Vec3(p)` constructor copy. Branch on `lenSq > 1e-16f` (was on `len > 1e-8f`) — same degenerate-plane safety, no extra cost. **Tests:** new "Frustum: all 6 planes have unit-length normals" pins the math invariant; "Frustum: sphere just touching far plane is treated consistently" pins the geometric consequence (sphere radius in same metric as world units). *trivial*.
- [ ] **FrustumCullSystem.cpp:41-43 + ShadowCullSystem.cpp:42-44** — `glm::abs` on `Vec4` then `Mat3` ctor copies 12 floats per entity. Replace with `fabsf` on the 9 used floats. **Saves ~30 ns per entity.** *trivial*.
- [ ] **FrustumCullSystem.cpp:50-55** — `reg.has<VisibleTag>` followed by `emplace/remove` is two sparse-set hits. Common case is "still visible" — store previous result in a parallel bit and only flip on change. **Saves ~40 ns × entity count.** *low*.
- [ ] **ShadowCullSystem.cpp:24-29, 51-69** — walks all mesh entities per cascade; 3 cascades = 3× iteration. Build single pass that tests all cascade frustums per entity. **Cuts shadow cull cost ~66% with 3 cascades.** *medium*.
- [ ] **DrawCallBuildSystem.cpp:165** — `reg.has<animation::SkinComponent>(entity)` per visible entity in the static-mesh loop just to skip skinned ones. Build view with explicit exclude (or a `StaticMeshTag`). **Saves ~25 ns × visible count.** *low*.
- [ ] **AnimationSystem.cpp:43-44, 318-319, 155, 337** — `std::pmr::vector<Mat4>(jointCount, Mat4(1.0f), alloc)` zero-fills bytes the inner loop overwrites immediately. Use uninitialised resize or skip fill. **Saves jointCount*16 stores per skinned entity.** *low*.
- [ ] **AnimationSystem.cpp:156-164** — `worldTransforms[i] = worldTransforms[parent] * local` full mat4*mat4. For pure TRS bones, concatenate in TRS-space then convert once = ~3× cheaper. **Big win with multiple characters.** *medium*.
- [ ] **AnimationSampler.cpp:79-93** — `sampleClip` re-initialises *every* joint to its rest pose, even joints not animated. Track which joints the clip touches and reset only those. **Saves jointCount*40 bytes write per frame per skinned entity.** *low*.
- [ ] **AnimationSampler.cpp:31-42, 58-67** — `glm::mix(Vec3,Vec3,t)` + `glm::slerp(Quat,Quat,t)` per channel. Skip channels whose two keyframes equal cached previous-frame result. *medium*.
- [ ] **PhysicsSystem.cpp:71-75, 137-138, 176-185** — `glm::decompose` does polar decomposition (QR iteration!) 3× per kinematic + 1× per parented dynamic per frame. Pass TRS directly via a `WorldTrsComponent` or cache decomposed-TRS alongside `WorldTransform`. **Saves 1-2 μs per physics entity.** *medium*.
- [ ] **PhysicsSystem.cpp:172** — `glm::inverse(parentWtc->matrix)` for every dynamic-body child every frame. Cache `inverseWorldMatrix` on `WorldTransformComponent`. **Saves ~80 muls per parented dynamic body.** *medium*.
- [ ] **AudioSystem.cpp:54-82** — `engine_.isPlaying()` round-trips into SoLoud's thread-locked voice table per entity. Batch: query live voice set once per frame, then iterate. **Avoids per-source mutex; halves audio update cost.** *medium*.

## Threading

- [~] **ThreadPool.cpp:27-41** — `submit()`/`waitAll()` mutex.  **Partial:** the long-held mutex pattern is gone — `submitTask` only holds `ringMutex_` for the few instructions that bump `tail_` and write the slot.  `waitAll()` spin-polls atomic `activeTasks_` for the first ~100 iterations before falling back to a CV wait; the CV is on a separate `doneMutex_` so it never contends with the ring.  **Still open:** per-worker queues + work-stealing for true MPMC scaling.  The current implementation should hit the audit's "<0.5 μs dispatch latency" target on the common path because the critical sections are now a handful of moves, but moving to per-worker queues would help when many submitters race.  *high refactor → partial.*
- [x] **(#H1 notify half) ThreadPool.cpp:67-70** — was `--activeTasks_` inside lock then unconditional `doneCv_.notify_one()`.  **Landed:** atomic `activeTasks_.fetch_sub(1, acq_rel)`; `doneCv_.notify_all()` fires only when the returned prior value is 1 (the transition-to-zero edge).  *low → landed.*
- [x] **(#H1 wakeup half) ThreadPool.cpp:54-61** — workers waiting on `workCv_` with the queue mutex held serialised wakeups.  **Landed:** `std::counting_semaphore<INT32_MAX>` for work-available.  `submitTask` does `workSem_.release()` outside the ring mutex; workers do `workSem_.acquire()` without holding any other lock.  Destructor releases `workers_.size()` tokens at once to drain on shutdown.  *medium → landed.*
- [x] **(#opt-in scheduler) No usage of `ThreadPool` for per-frame work** — `TransformSystem`, `FrustumCullSystem`, `DrawCallBuildSystem`, `ShadowCullSystem`, `LightClusterBuilder` all ran single-threaded on the game thread.  **Landed (opt-in scaffolding):** new `EngineDesc::useSystemThreadPool` (default **false** — historical behaviour preserved) constructs an Engine-owned `threading::ThreadPool`.  `Engine::systemThreadPool()` returns a pointer for game-side use (nullptr when opt-in disabled).  `engine/ecs/SystemExecutor.h` was already a templated phase-based dispatcher but had no users — it now goes through the new POD `submitTask` path and has 7 unit tests covering single-system inline dispatch, multi-system phase parallelism (timing-sensitive), phase-ordering barriers, getSystem accessor, and a 10 000-frame conservation-law race-check that's TSAN-friendly.  Engine does NOT route any built-in system through the pool — what to parallelise is per-game.  Tier-aware pool sizing via `EngineDesc::systemThreadPoolSize` (0 = engine picks via `hardware_concurrency() - 2` clamped to [2, 8]).  *high refactor (experiment) → landed (opt-in scaffolding); the per-system phase declarations for the actual built-in systems are still future work, gated on per-tier device measurement.*

## Rendering / bgfx

- [x] ⭐ **(#R1) DrawCallBuildSystem.cpp:181-190, 401-410** — `u_dirLight`, `u_shadowMatrix`, `u_frameParams`, `u_lightParams`, `u_iblParams` are all **frame-constant** but uploaded *per draw call*. **Landed via Encoder route, not uniform-hoist.** Investigation: the audit suggested using `BGFX_DISCARD_NONE`-style submit flags to preserve uniforms across draws, but bgfx captures `[m_uniformBegin, m_uniformEnd]` per draw and the kept range grows unboundedly — O(N²) GPU-side uniform processing per N-entity frame, which is *worse* than the original. The actual win comes from acquiring `bgfx::begin()` once per system call: every set*/submit then runs thread-local (~50 ns) instead of grabbing the bgfx command-list mutex (~1-2 μs on Pixel 5a-class CPUs). For ~1000 set/submit calls per frame in `DrawCallBuildSystem` + `submitShadowDrawCalls` + skinned variants, expected saving on Android ~1-2 ms / frame. **Desktop M-series shows no measurable delta** (mutex is uncontested locally on MoltenVK). Also fixed: hoisted IBL fallback texture resolution outside the loop (was 3 `bgfx::isValid` checks per draw). Tests: 716 unit + 22 screenshot still pass — Encoder API is functionally identical. **Measured on Android emulator (sama_high AVD, 702-entity perf_smoke scene, 600 frames, A/B same session via `git revert` + rebuild + reinstall): DrawCallBuildSystem mean 0.043 → 0.042 ms (flat within noise), p99 0.167 → 0.116 ms (−30%), max 0.348 → 0.174 ms (−50%).** Tail-latency drop is the signature of removed mutex contention — mean is flat because uncontested mutex grabs are fast, p99/max capture the times another thread held it. On real Pixel 5a-class ARM hardware the mean would also drop because the emulator's x86_64-translation has lower native mutex cost than the target. *low.*

Cannot exercise the actual Mali fp16 win from a Mac desktop test; integration validation will require an Android emulator + Pixel-class device frame trace.  *medium (banding-risk).*  Headline-list count of "7+" was actually 5 (1× `s_lightIndex` + 4× `s_lightData` per light); precision win lands on all of them plus the ~5 fixed material samples.
